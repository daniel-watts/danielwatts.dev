## What this is

A single-page resume/portfolio website for danielwatts.dev, built with [Astro](https://astro.build) v6. It ships zero JavaScript by default and builds to plain static HTML in `./dist/`. There is no client-side framework, no database, and no backend — it is a static site.

## Commands

Uses Node via `nvm`. If `node` isn't found, run `nvm use` first.

| Command | Action |
| --- | --- |
| `npm install` | Install dependencies |
| `npm run dev` | Dev server with hot reload at `localhost:4321` |
| `npm run build` | Build static output to `./dist/` |
| `npm run preview` | Serve the production build locally |

There is no test suite, linter, or formatter configured. TypeScript uses Astro's `strict` config; type errors surface during `npm run build`.

### Docker

- Production (nginx serving static build): `docker compose up` → `localhost:8080`
- Dev (hot reload in container): `docker compose --profile dev up dev` → `localhost:4321`

The Dockerfile is multi-stage: `build` (Node, runs `npm run build`) → `runtime` (nginx serving `/dist`), plus a `dev` stage for the compose dev profile.

## Architecture

**Content is fully decoupled from presentation.** All site content — name, bio, experience, projects, skills, contact links, SEO description — lives in `src/data/site.ts`, exported as typed objects (`site`, `hero`, `about`, `experience`, `projects`, `skills`, `contact`). For normal content changes, edit this file only; do not touch components.

**Rendering flow:**
- `src/pages/index.astro` is the only page. It composes section components (`Hero`, `About`, `Experience`, `Projects`, `Skills`, `Contact`) plus `Header`/`Footer` inside `BaseLayout`.
- `src/layouts/BaseLayout.astro` owns the `<head>`: title/description, canonical URL, Open Graph + Twitter cards, and JSON-LD `Person` structured data — all derived from `site.ts`.
- Each component in `src/components/` imports the relevant object from `site.ts` and renders it. Components are presentational; they don't define content.
- `src/styles/global.css` holds all styling, including the light/dark theme (follows system preference) and the brand color via the `--color-accent` CSS variable.

The sitemap is auto-generated by `@astrojs/sitemap` (configured in `astro.config.mjs`).

## When editing

- **Changing content** (jobs, projects, links, bio): edit `src/data/site.ts`.
- **Changing the production URL**: update both `astro.config.mjs` (`site`) and `site.url` in `src/data/site.ts` — they must match, since canonical/OG/sitemap URLs derive from them.
- **Changing brand color**: `--color-accent` in `src/styles/global.css`.
- **Static assets** (favicon, OG image, robots.txt): `public/`.
- `dist/` is build output — never edit it by hand.

For an Astro primer, see `docs/astro-overview.md`.
